# 医院内部绩效考核与病历协作系统操作使用说明

> 文档版本：V1.0  
> 生成日期：2026-05-03  
> 适用对象：超级管理员、院领导、项目负责人/板块负责人、普通员工  
> 使用范围：本院内部绩效考核、周任务闭环、整改跟踪、病历协作填写 MVP

---

## 1. 文档目的

本说明用于帮助院内不同角色快速理解系统能做什么、每天怎么用、遇到问题怎么处理。

系统当前主要解决四类问题：

| 模块 | 解决的问题 | 主要使用人 |
| --- | --- | --- |
| 绩效考核 | 把全员通用考核、板块专项考核、周期提交、审核确认放到一个系统里 | 员工、负责人、院领导 |
| 本周任务 | 负责人下发任务，员工完成后上传佐证材料，负责人审核 | 负责人、员工 |
| 整改台账 | 对退回、未达标、需整改事项进行闭环跟踪 | 员工、负责人、院领导 |
| 病历协作 | 按岗位分段填写病历信息，右侧实时预览，最终导出 | 前台、护理、医生、检查、手术、出院整理、审核人员 |

## 3. 角色与权限速查

| 角色 | 主要菜单 | 能做什么 | 不能做什么 |
| --- | --- | --- | --- |
| 超级管理员 | 系统管理、考核平台、病历协作 | 新增账号、编辑资料、重置密码、查看账号日志、配置权限、配置组织责任、设置周期、查看和处理全部数据 | 不建议替代员工长期代填日常内容 |
| 院领导 | 考核平台、病历协作、汇总看板、整改台账 | 查看全院进度、风险、排名、整改闭环，进行领导层确认 | 不负责日常逐项录入 |
| 项目负责人/板块负责人 | 考核平台、审核台、本周任务、整改台账、病历协作 | 下发任务、审核员工提交、退回整改、协助录入、确认本板块结果 | 默认只能处理本板块或授权范围 |
| 普通员工 | 我的考核、我的任务、病历填写、我的整改、我的结果 | 填写本人考核、完成任务并上传佐证、处理整改、确认最终结果 | 不能查看其他员工隐私数据，不能修改未授权病历字段 |

## 4. 系统主流程

### 4.1 绩效考核主流程

```text
超级管理员配置账号和周期
→ 员工填写本周期考核项
→ 员工提交本周期考核
→ 负责人审核
→ 如有问题退回整改
→ 员工补充或整改后再次提交
→ 负责人确认本板块结果
→ 院领导查看汇总并确认
→ 员工查看并确认本人结果
```

### 4.2 任务闭环主流程

```text
负责人创建任务
→ 推送给指定员工或板块人员
→ 员工在“我的任务”查看任务
→ 员工上传至少 1 份佐证材料
→ 员工标记任务已完成
→ 负责人在审核台查看说明和附件
→ 负责人通过或退回
→ 退回后员工补充材料再次提交
```

### 4.3 病历协作主流程

```text
前台建档
→ 护理填写基础护理信息
→ 检查岗位录入检查结果
→ 医生填写主诉、诊断、治疗意见
→ 手术岗位填写手术相关信息
→ 出院整理人员补充出院信息
→ 负责人审核
→ 导出或打印预览病历
```

## 5. 超级管理员操作说明

超级管理员是系统维护人员，主要负责“账号、权限、组织、周期、基础配置”。

### 5.1 新增员工账号

入口：

```text
系统管理 → 新增用户
```

操作步骤：

1. 填写登录账号。
2. 填写初始密码。
3. 填写姓名、工号、邮箱、手机号。
4. 选择所属板块。
5. 填写岗位名称。
6. 按实际工作选择“系统权限”。
7. 如该员工参与病历协作，选择“病历权限”。
8. 点击“保存用户”。

字段说明：

| 字段     | 说明                                         |
| -------- | -------------------------------------------- |
| 登录账号 | 员工登录系统使用，建议使用姓名拼音或工号     |
| 初始密码 | 首次登录使用，后续可由管理员重置             |
| 所属板块 | 用于确定该员工属于哪个考核和审核范围         |
| 岗位     | 用于展示岗位身份，便于负责人识别             |
| 系统权限 | 决定账号是超级管理员、院领导、负责人还是员工 |
| 病历权限 | 决定该账号能填写病历的哪些板块               |
| 友好模式 | 对不熟悉系统的员工提供更清晰的提示           |

> 【截图建议 3：新增用户页面】  
> 建议截图内容：重点标出“所属板块”“系统权限”“病历权限”三个字段。

### 5.2 编辑员工资料和重置密码

入口：

```text
系统管理 → 员工管理 → 编辑资料
```

可维护内容：

| 内容                     | 用途                         |
| ------------------------ | ---------------------------- |
| 登录账号                 | 修改员工登录名               |
| 密码                     | 修改或重置员工密码           |
| 姓名、工号、邮箱、手机号 | 员工基础信息维护             |
| 所属板块                 | 决定考核归属和负责人审核范围 |
| 岗位                     | 决定实际岗位展示             |
| 系统权限                 | 决定菜单和数据范围           |
| 病历权限                 | 决定病历协作可编辑字段       |
| 账号状态                 | 启用或停用账号               |

建议规则：

| 场景               | 建议处理                         |
| ------------------ | -------------------------------- |
| 新员工入职         | 新增账号，绑定板块和岗位         |
| 员工调岗           | 修改所属板块、岗位、病历权限     |
| 员工忘记密码       | 在员工管理中重置密码             |
| 员工离职或暂不使用 | 停用账号，不建议直接删除历史数据 |

### 5.3 查看账号日志

入口：

```text
系统管理 → 账号日志
```

账号日志用于查看：

| 内容       | 说明                   |
| ---------- | ---------------------- |
| 登录账号   | 哪个账号尝试登录       |
| 登录结果   | 成功或失败             |
| 登录时间   | 具体登录时间           |
| IP 地址    | 登录来源地址           |
| 浏览器信息 | 使用的浏览器和设备信息 |
| 失败原因   | 账号不存在、密码错误等 |

使用建议：

1. 如果员工反馈无法登录，先查看账号日志。
2. 如果频繁出现密码错误，可以重置密码。
3. 如果出现陌生 IP 频繁尝试，建议临时停用账号并排查。

### 5.4 权限分配

入口：

```text
系统管理 → 权限分配
```

该页面用于维护角色权限。正常情况下不建议频繁修改，除非系统流程确实调整。

角色建议：

| 角色       | 建议分配对象                           |
| ---------- | -------------------------------------- |
| 超级管理员 | 系统维护人员，建议 1 至 2 人           |
| 院领导     | 院长、副院长、需要查看全院数据的管理层 |
| 板块负责人 | 医疗、护理、院感、行政后勤等板块负责人 |
| 普通员工   | 大多数一线员工                         |

### 5.5 组织责任配置

入口：

```text
系统管理 → 组织责任配置
```

该页面用于把“板块”和“负责人”绑定起来。

配置后会影响：

| 影响点         | 说明                           |
| -------------- | ------------------------------ |
| 负责人审核范围 | 负责人主要看到自己板块员工     |
| 任务下发范围   | 负责人可面向本板块人员安排任务 |
| 汇总看板       | 板块统计按组织责任归类         |
| 整改台账       | 整改事项可以按板块追踪         |

操作建议：

1. 先确认院内板块划分。
2. 再确定每个板块的分管领导、负责人、协同人员。
3. 最后再创建或调整员工账号。

### 5.6 定编定岗

入口：

```text
系统管理 → 定编定岗
```

该模块用于维护岗位架构、岗位人数、核心负责人和岗位职责。

适合用于：

| 场景         | 说明                                             |
| ------------ | ------------------------------------------------ |
| 梳理院内岗位 | 对照领导提供的定编定岗文件整理岗位               |
| 明确责任链   | 明确每个岗位向谁汇报                             |
| 汇报展示     | 向领导说明系统不是单纯打分，而是围绕岗位职责落地 |

### 5.7 周期模板

入口：

```text
系统管理 → 周期模板
```

周期模板用于控制当前考核周期和提交截止时间。

建议流程：

1. 每周或每月开始前，由超级管理员确认本周期。
2. 设置员工提交截止时间。
3. 设置负责人审核截止时间。
4. 周期结束后查看完成情况和未确认人员。

## 6. 院领导端操作说明

院领导端主要用于看全局，不建议承担大量日常录入。

### 6.1 查看考核工作台

入口：

```text
考核平台 → 考核工作台
```

建议关注：

| 指标       | 关注点                               |
| ---------- | ------------------------------------ |
| 当前周期   | 是否是正在执行的周期                 |
| 完成数量   | 员工是否按时填写                     |
| 未完成事项 | 哪些板块或人员滞后                   |
| 风险提示   | 是否存在较多未整改、未确认、低分事项 |

### 6.2 查看汇总看板

入口：

```text
考核平台 → 汇总看板
```

适合领导查看：

| 内容       | 用途                 |
| ---------- | -------------------- |
| 全院完成率 | 判断系统推进情况     |
| 板块得分   | 横向比较不同板块表现 |
| 低分人员   | 找出需要重点关注人员 |
| 整改闭环   | 查看整改是否真正完成 |
| 确认状态   | 判断周期能否归档     |

> 【截图建议 4：院领导汇总看板】  
> 建议截图内容：全院完成率、板块排名、风险提示、整改闭环。

### 6.3 查看整改台账

入口：

```text
考核平台 → 整改台账
```

院领导可用于追踪：

| 问题           | 查看方式             |
| -------------- | -------------------- |
| 哪些问题被退回 | 查看退回记录         |
| 谁负责整改     | 查看责任人和所属板块 |
| 是否完成闭环   | 查看整改状态         |
| 是否反复出现   | 对比不同周期记录     |

### 6.4 领导确认

入口通常在：

```text
考核平台 → 汇总看板
```

建议在以下条件满足后再进行确认：

1. 大多数员工已提交。
2. 负责人已完成主要审核。
3. 关键整改问题已经有处理意见。
4. 低分或异常事项已经被负责人说明。
5. 本周期结果可以用于内部管理复盘。

## 7. 项目负责人/板块负责人操作说明

板块负责人是系统日常运行的核心角色，主要负责“任务安排、审核、退回、整改闭环、板块确认”。

### 7.1 查看本板块员工进度

入口：

```text
考核平台 → 考核工作台
考核平台 → 汇总看板
```

负责人需要重点看：

| 内容           | 说明                       |
| -------------- | -------------------------- |
| 员工提交情况   | 哪些人还没有提交           |
| 考核项完成情况 | 哪些项目未完成或有备注     |
| 任务完成情况   | 哪些任务未完成、未上传佐证 |
| 整改情况       | 哪些问题退回后仍未处理     |

### 7.2 下发本周任务

入口：

```text
考核平台 → 本周任务
```

操作步骤：

1. 点击新增或编辑任务。
2. 填写任务名称和任务说明。
3. 选择任务对象。
4. 设置截止时间。
5. 保存并推送。

任务对象可以是：

| 对象         | 说明                       |
| ------------ | -------------------------- |
| 指定员工     | 只推给某几个人             |
| 指定类别人员 | 推给某个板块或岗位相关人员 |
| 本板块员工   | 面向负责人所管辖范围       |

注意事项：

1. 新任务推送后默认是“未完成”。
2. 员工不能直接空点完成，必须上传至少 1 份佐证材料。
3. 如果任务超过截止时间，员工端可能会提示锁定或需要负责人处理。

### 7.3 审核员工提交

入口：

```text
考核平台 → 审核台
```

审核台会显示员工提交的事项，包括考核项和任务。

负责人处理方式：

| 操作 | 适用情况                               |
| ---- | -------------------------------------- |
| 通过 | 员工填写真实完整，佐证材料符合要求     |
| 退回 | 信息不完整、材料不清楚、结果不符合要求 |

审核任务时需要重点查看：

1. 员工补充说明。
2. 上传的佐证附件。
3. 附件是否与任务内容对应。
4. 是否存在只上传无关文件凑数的情况。

### 7.4 协助录入

入口：

```text
考核平台 → 负责人视图/员工进度相关页面 → 协助录入
```

协助录入适合以下情况：

| 场景             | 说明                         |
| ---------------- | ---------------------------- |
| 员工不会操作     | 负责人可帮助录入基础内容     |
| 员工临时无法填写 | 负责人可先代录，避免周期卡住 |
| 纸面材料已提交   | 负责人可根据纸面材料录入     |

重要规则：

1. 协助录入会写入审核或操作记录。
2. 协助录入不替代员工本人最终确认。
3. 员工仍需进入“我的结果”完成电子确认。

### 7.5 板块结果确认

负责人应在以下事项完成后再确认：

1. 本板块员工考核基本完成。
2. 任务审核基本处理完毕。
3. 退回整改有明确处理结果。
4. 异常低分或特殊情况已经备注。
5. 员工最终确认状态已经尽量补齐。

## 8. 普通员工端操作说明

普通员工端重点是“少看复杂数据，只完成自己该做的事”。

### 8.1 登录后先看哪里

员工登录后主要使用以下菜单：

| 菜单     | 用途                                 |
| -------- | ------------------------------------ |
| 我的考核 | 填写本周期考核内容                   |
| 我的任务 | 查看负责人安排的任务，上传佐证并完成 |
| 病历填写 | 按岗位权限填写病历内容               |
| 我的整改 | 查看被退回或要求整改的问题           |
| 我的结果 | 查看最终结果并电子确认               |

> 【截图建议 6：员工端首页】  
> 建议截图内容：我的考核、我的任务、我的整改、我的结果四个入口。

### 8.2 填写我的考核

入口：

```text
员工端 → 我的考核
```

操作步骤：

1. 查看当前周期。
2. 按页面提示检查考核项。
3. 对每项选择完成状态。
4. 必要时填写备注或整改说明。
5. 全部确认后提交本周期。

填写建议：

| 情况         | 建议                     |
| ------------ | ------------------------ |
| 确实已完成   | 选择已完成               |
| 暂未完成     | 保持未完成，并填写原因   |
| 有特殊情况   | 在备注中说明             |
| 被负责人退回 | 按退回意见补充后再次提交 |

### 8.3 完成我的任务

入口：

```text
员工端 → 我的任务
```

任务完成规则：

1. 任务默认是“未完成”。
2. 完成任务前必须上传至少 1 份佐证材料。
3. 文字说明只是补充，不能替代文件。
4. 上传成功后才能标记为“已完成”。
5. 标记完成后，负责人会在审核台查看并处理。

支持上传的材料类型：

| 类型   | 示例               |
| ------ | ------------------ |
| 图片   | 工作现场照片、截图 |
| PDF    | 扫描件、报告       |
| Word   | 说明材料、记录文档 |
| Excel  | 台账、统计表       |
| PPT    | 汇报材料           |
| 压缩包 | 多个材料打包       |

单个文件建议不超过 20MB。

操作步骤：

1. 打开“我的任务”。
2. 找到需要处理的任务。
3. 在“完成佐证”区域填写补充说明。
4. 点击“上传材料”。
5. 上传至少 1 个真实文件。
6. 点击“标记为已完成”。
7. 等待负责人审核。

> 【截图建议 7：员工任务佐证上传】  
> 建议截图内容：补充说明输入框、上传材料按钮、附件列表、标记完成按钮。

### 8.4 处理我的整改

入口：

```text
员工端 → 我的整改
```

整改来源可能包括：

| 来源       | 说明                     |
| ---------- | ------------------------ |
| 考核项退回 | 填写不完整或不符合要求   |
| 任务退回   | 任务材料不足或结果不明确 |
| 负责人要求 | 负责人认为需要补充整改   |

处理步骤：

1. 查看整改原因。
2. 根据负责人意见补充内容。
3. 如果涉及任务，补充上传材料。
4. 再次提交或等待负责人复核。

### 8.5 查看并确认我的结果

入口：

```text
员工端 → 我的结果
```

员工需要查看：

| 内容       | 说明                     |
| ---------- | ------------------------ |
| 本周期得分 | 当前周期绩效结果         |
| 完成数量   | 已完成项目数量           |
| 整改记录   | 是否有未闭环事项         |
| 确认状态   | 是否已经完成本人电子确认 |

确认前建议：

1. 检查本人考核项是否准确。
2. 检查任务审核结果是否正确。
3. 如有异议，先联系负责人处理。
4. 无异议后点击确认。

确认后原则上不再随意修改。

## 9. 病历协作操作说明

病历协作是当前系统的 MVP 功能，目标是把原本“多人填写同一个表格”的过程改成“按岗位分段填写，右侧实时预览最终文档”。

### 9.1 页面结构

入口：

```text
员工端 → 病历填写
考核平台 → 病历协作
```

页面分为三块：

| 区域     | 用途                           |
| -------- | ------------------------------ |
| 左侧列表 | 查看已有病历，选择要编辑的病历 |
| 中间表单 | 按岗位权限填写字段             |
| 右侧预览 | 实时查看最终导出病历效果       |

> 【截图建议 8：病历协作页面全景】  
> 建议截图内容：左侧病历列表、中间填写表单、右侧实时预览。

### 9.2 病历权限如何控制

病历填写权限由超级管理员在账号中配置。

入口：

```text
系统管理 → 新增用户/员工管理 → 病历权限
```

可选权限：

| 病历权限   | 适合岗位             | 可填写内容                               |
| ---------- | -------------------- | ---------------------------------------- |
| 前台建档   | 前台、登记人员       | 患者代号、住院号尾号、入院时间等基础信息 |
| 护理填写   | 护理人员             | 生命体征、护理观察、护理相关内容         |
| 医生填写   | 医生                 | 主诉、专科检查、中西医诊断、治疗意见     |
| 检查录入   | 检查相关人员         | 肠镜、检验、辅助检查结果                 |
| 手术记录   | 手术相关人员         | 手术日期、术式、术中情况                 |
| 出院整理   | 出院整理人员         | 出院诊断、出院医嘱、复诊建议             |
| 负责人审核 | 医疗负责人、审核人员 | 审核状态和最终确认                       |

超级管理员默认可以编辑全部字段。

普通员工只能编辑自己被授权的字段，其他字段会显示为只读。

### 9.3 新建病历

通常由前台或有“前台建档”权限的人员新建。

操作步骤：

1. 打开“病历填写”。
2. 点击“新建病历”。
3. 填写患者代号、住院号尾号等基础信息。
4. 不建议在 MVP 阶段直接填写完整真实身份信息。
5. 点击“保存”。

敏感信息建议：

| 信息     | 当前建议                         |
| -------- | -------------------------------- |
| 患者姓名 | 使用患者代号，不直接显示真实姓名 |
| 身份证号 | 不录入或只录入脱敏信息           |
| 联系电话 | 不录入或只录入必要尾号           |
| 住院号   | 可使用尾号或内部识别号           |

### 9.4 分岗位填写

各岗位只需要填写自己负责的板块。

操作方式：

1. 选择左侧病历。
2. 查看中间表单中可编辑字段。
3. 能编辑的字段正常填写。
4. 显示“只读”的字段由其他岗位填写。
5. 填写后点击“保存”。
6. 右侧预览会同步更新。

表单控件说明：

| 控件         | 用法                             |
| ------------ | -------------------------------- |
| 文本框       | 填写主诉、说明、诊断等长文本     |
| 选择器       | 选择固定选项，减少手工输入错误   |
| 日期时间选择 | 选择入院、手术、出院等时间       |
| 数字输入     | 填写体温、脉搏、检查数值等       |
| 多选项       | 选择多个症状、处理方式或注意事项 |

### 9.5 中西医诊断填写

中西医诊断不是完全模板化内容，必须由医生结合患者实际指征判断。

当前建议：

1. 诊断字段保留为医生可自由填写。
2. 可参考常见诊断模板，但不能机械套用。
3. 如需要标准化，后续可增加“常用诊断短语库”。
4. 最终导出前必须由医生或负责人确认。

### 9.6 提交审核

通常由医生或负责人在主要内容填写完成后提交。

操作步骤：

1. 检查右侧预览是否完整。
2. 检查关键诊断、手术、出院信息是否缺漏。
3. 点击“提交审核”。
4. 提交后状态变为待审核。

### 9.7 导出或打印病历

入口：

```text
病历协作 → 选择病历 → 导出 / 打印
```

当前导出逻辑：

1. 系统会打开单独的预览打印页面。
2. 打印内容只包含右侧预览病历，不包含左侧列表和中间表单。
3. 可以在浏览器打印窗口中选择“另存为 PDF”。
4. 正式病历建议经医生确认后再导出。

> 【截图建议 9：病历预览导出页】  
> 建议截图内容：只显示病历抬头、单位名称、正文预览，不显示操作按钮。

## 10. 常见问题处理

### 10.1 登录失败

可能原因：

| 原因           | 处理                 |
| -------------- | -------------------- |
| 账号输错       | 核对账号             |
| 密码输错       | 联系管理员重置密码   |
| 账号被停用     | 联系管理员启用       |
| 后端服务未启动 | 联系维护人员启动服务 |

管理员排查入口：

```text
系统管理 → 账号日志
```

### 10.2 页面提示网络未连接

常见原因：

1. 后端 API 服务没有启动。
2. 前端页面打开了，但后端端口未运行。
3. 电脑或服务器网络变化导致局域网地址变化。
4. 浏览器缓存了旧页面。

处理建议：

1. 先确认前端和后端都已启动。
2. 刷新浏览器页面。
3. 使用当前电脑新的局域网 IP 地址访问。
4. 如果仍不行，让维护人员检查服务端口。

### 10.3 员工看不到某个菜单

原因通常是角色不同。

| 现象                   | 说明                         |
| ---------------------- | ---------------------------- |
| 员工看不到系统管理     | 正常，员工无管理员权限       |
| 员工看不到审核台       | 正常，员工不是负责人         |
| 负责人看不到员工端菜单 | 正常，负责人主要使用考核平台 |
| 某员工不能填写病历字段 | 需要管理员检查“病历权限”     |

### 10.4 员工任务无法标记完成

原因通常是没有上传佐证材料。

处理方式：

1. 打开“我的任务”。
2. 在任务卡片中找到“完成佐证”。
3. 上传至少 1 个真实文件。
4. 再点击“标记为已完成”。

### 10.5 上传材料失败

可能原因：

| 原因           | 处理                                |
| -------------- | ----------------------------------- |
| 文件超过 20MB  | 压缩后重新上传                      |
| 文件类型不支持 | 转为 PDF、图片、Office 文档或压缩包 |
| 网络中断       | 刷新后重新上传                      |
| 后端服务异常   | 联系维护人员检查服务                |

### 10.6 病历切换页面后未保存

当前系统已有自动保存和本机临时草稿机制，但仍建议：

1. 关键字段填写后及时点击“保存”。
2. 切换菜单前先确认页面提示已保存。
3. 如果提示网络异常，先不要关闭页面。
4. 网络恢复后再点击保存。

### 10.7 导出病历时浏览器没有反应

可能是浏览器拦截了弹窗。

处理方式：

1. 查看地址栏右侧是否有弹窗拦截提示。
2. 允许当前网站弹出窗口。
3. 再点击“导出 / 打印”。

## 12. 汇报演示建议

如果晚上向领导汇报，建议按以下顺序演示：

1. 登录页：说明系统是院内绩效考核和病历协作统一入口。
2. 超级管理员新增用户：说明能按岗位、板块、病历权限分配账号。
3. 员工端我的考核：说明普通员工只看到自己要处理的事项。
4. 负责人下发任务：说明任务能推送到指定人员或板块。
5. 员工上传任务佐证：说明“完成”不再是空点，必须有真实材料。
6. 负责人审核台：说明负责人能看到说明和附件，可以通过或退回。
7. 汇总看板：说明院领导能看全院进度、风险和整改闭环。
8. 病历协作：说明不同岗位分段填写，右侧实时预览，最后单独导出病历。
9. 账号日志：说明系统可以追踪账号登录情况。
10. 定编定岗：说明系统与领导提供的岗位架构文件能够逐步对齐。

建议汇报重点：

| 重点               | 说明                                                   |
| ------------------ | ------------------------------------------------------ |
| 不是单纯打分       | 系统把岗位、任务、佐证、整改和确认串起来               |
| 普通员工学习成本低 | 员工只需要处理“我的考核、我的任务、我的整改、我的结果” |
| 负责人有抓手       | 下发任务、看佐证、退回整改都有流程                     |
| 院领导看结果       | 汇总看板能看进度、风险和闭环                           |
| 病历协作可扩展     | 当前是 MVP，后续可继续加入模板库、审核签名、权限细化   |
