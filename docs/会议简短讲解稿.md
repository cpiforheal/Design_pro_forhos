# 医院内部绩效与病历协作系统简短讲解稿

> 适合会议现场 3 到 5 分钟介绍。  
> 本次主要讲三个角度：普通员工、板块负责人、病历填写协作。

## 一、开场说明

各位领导、同事，今天简单汇报一下当前这个系统的核心使用方式。

这个系统的目标不是单纯做一个展示页面，而是把我们原来分散在 Word、表格、人工通知和线下汇总里的工作，尽量收敛到一个可以操作、可以追踪、可以检查的平台里。

当前系统主要解决三件事：

- 普通员工知道自己每周该做什么、哪些事项需要完成、哪些需要补材料。
- 板块负责人可以安排本板块任务，并查看本板块人员完成情况。
- 病历填写可以由不同岗位分段填写，最后交由指定医师审核归档。

## 二、普通员工视角

从普通员工角度看，登录系统之后，重点不需要到处找功能，主要看“我的任务”或“我的待办”。

这里会把员工本人需要处理的事项集中展示出来，包括全员通用考核、本岗位专项考核、医院安排任务、负责人安排任务等。不同来源的任务会用不同标签区分，员工可以通过筛选快速查看自己当前要处理的内容。

对于普通员工来说，系统的使用逻辑比较简单：

1. 登录后进入首页，先查看自己的待办和考核情况。
2. 点击“去处理”，进入对应事项。
3. 已完成的任务，需要上传佐证材料，比如图片、PDF、Word、Excel 等文件。
4. 提交后，负责人可以查看完成情况和佐证材料。

这样做的好处是，员工不需要再靠微信群、口头通知或纸质表格去记任务，系统会把“该做什么、是否完成、有没有材料”集中呈现出来。

## 三、板块负责人视角

板块负责人比普通员工多一层管理职责。

负责人除了完成自己作为员工的考核事项外，还需要对本板块的工作进行安排、跟踪和审核。

目前负责人端主要有几个核心动作：

1. 查看本板块员工的任务完成情况。
2. 新增本板块分管工作，比如周一重点任务、临时突击任务、负责人安排任务等。
3. 对员工提交的任务和佐证材料进行审核。
4. 对未完成或存在问题的事项进行退回、整改或继续跟踪。

这里的设计重点是“各岗位只干各自该干的活”。普通员工只看自己的任务，板块负责人看本板块的任务和人员完成情况，院长端再从全院角度做监管。

因此，负责人不需要再手工统计每个人完成了什么，也不需要反复催材料。系统里可以直接看到完成、未完成、材料提交和审核状态。

## 四、病历填写协作视角

病历协作部分，是针对我们现在病历信息分散、填写不规范、表格不好用的问题做的一个最小可用版本。

它的基本思路是：一份病历不是由一个人一次性填完，而是按照患者流程，由不同岗位分段填写。

比如：

- 前台或接诊岗位负责创建病历，填写基础登记信息。
- 护理岗位填写护理相关内容。
- 医师填写诊断、治疗方案、手术及专业判断内容。
- 检查、手术、出院等环节由对应岗位补充信息。
- 最后提交给指定医师审核归档。

页面设计上，左侧是填写表单，右侧是实时预览。填写人员可以边填边看最终病历呈现效果，避免最后导出时格式混乱。

这里也做了基本岗位权限控制，不同岗位只能填写自己负责的部分。超级管理员可以指定最终审核医师，病历提交后会收敛到指定医师那里进行确认。

这样可以把原来“大家都在一个表格里改、容易漏填、格式不统一”的问题，转成一个更清晰的流水线流程。

## 五、总结

整体来说，当前系统的核心价值可以概括为三句话：

- 对普通员工：减少理解成本，只看自己要完成的任务。
- 对板块负责人：减少人工统计成本，直接跟踪本板块工作完成情况。
- 对病历协作：减少表格混乱和重复沟通，让不同岗位按流程填写，最后统一审核归档。

当前版本还没有把上线部署和项目包体积问题完全解决，所以今天更适合作为业务流程和功能原型进行演示。后续如果确认方向可行，再继续处理服务器部署、访问地址、数据备份和正式上线这些问题。
